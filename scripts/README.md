# Scripts

Two categories: canonical-chain steps that produce the paper's locked artifacts, and figure-data scripts that consume those artifacts to produce figure-specific data.

## Canonical chain (four steps)

These run in order and produce the paper's cited CSV outputs and the methodology manifest. Each step depends on the prior step's outputs.

```
node scripts/preprocess-bitstamp.js
node scripts/export.js
node scripts/build-surface.js
node scripts/build-surface-grid-validation.js
```

| Step | Script | Reads | Writes |
|---|---|---|---|
| 1 | `preprocess-bitstamp.js` | `data/raw/bitstamp_btc_prices.csv` | `data/btc-prices-bitstamp.csv`, `data/btc-prices-bitstamp-imputation.json` |
| 2 | `export.js` | `data/btc-prices-bitstamp.csv`, `data/btc-prices-coinmetrics.csv` | `outputs/manifest.json` (initial), `outputs/regime_summary.csv`, `outputs/regime_descriptors.csv`, `outputs/bcr_paths_monthly.csv`, `outputs/bcr_paths_daily.csv`, `outputs/transition_paths_monthly.csv` |
| 3 | `build-surface.js` | step 2 outputs | `outputs/regime_tau_sigma.csv`, `outputs/time_at_depth_distributions.csv`, `outputs/surface_validation.csv`, `outputs/surface_grid.csv`; merges `phase_2` block into `outputs/manifest.json` |
| 4 | `build-surface-grid-validation.js` | step 3 outputs (manifest pooled (τ, σ); regime_descriptors.csv) | `outputs/surface_grid_validation.csv`; writes `phase_2.sha256.surface_grid_validation` into the manifest. §6.6.2 body comparison under grid calendar start (2017-12-16); does not rebuild the surface |

### Daily-cadence variant (paper §5.1 footnote 1)

The daily run only needs steps 1 and 2; there is no daily surface. The surface and its body-table validation are constructed only at monthly cadence per paper §6.6.

```
node scripts/preprocess-bitstamp.js
node scripts/export.js --cadence=daily
```

Outputs land in `outputs/daily/`. The `--cadence=daily` flag is honored by `export.js` only. `preprocess-bitstamp.js`, `build-surface.js`, and `build-surface-grid-validation.js` ignore the flag.

### `lib/` modules

`lib/engine.js`, `lib/runner.js`, `lib/aggregator.js`, `lib/monthly.js`, `lib/prices.js`, `lib/regimes.js`, `lib/stats.js`, `lib/writers.js`. Internal helpers loaded by step 2; not invoked directly.

## Figure data script

### `derive-mech-chart-data.js`

Folded into `export.js`'s `main()` as the final step. Streams `outputs/bcr_paths_daily.csv` line-by-line (the file is ~750 MB at monthly cadence, ~850 MB at daily) and emits per-scenario minimum mechanical-continuation BCR. Drives Figure 2 (four main regimes) and Figure B1 (2011 regime, Appendix B). Writes `outputs/min_bcr_chart_data_mech.csv` (or `outputs/daily/min_bcr_chart_data_mech.csv`); the canonical chain regenerates this in lockstep.

Remains standalone-runnable for one-off rebuilds, including a `--include=covid` flag that adds covid_2020 (not regenerated by the canonical chain):

```
node scripts/derive-mech-chart-data.js                 # monthly cadence
node scripts/derive-mech-chart-data.js --cadence=daily # daily cadence
node scripts/derive-mech-chart-data.js --include=covid # 6 regimes incl. covid_2020
```

The canonical paper figures (`figures/fig01`–`fig07`, `figB1`, `figB2`) are mirrored into this repo from the clean-room visualization workspace; they are not regenerated by any script in this directory.
